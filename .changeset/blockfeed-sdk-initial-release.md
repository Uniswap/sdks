Initial release: block-latency on-chain data feeds over any viem `PublicClient`. `createBlockFeed` runs one refcounted atomic heartbeat per `(client, chainId)` — a single Multicall3 batch (block number/hash/timestamp + every active source's state reads, deduped and chunked) plus one trailing-window `eth_getLogs` per tick — over both HTTP (polled) and WebSocket (push-triggered) transports; `chainId` defaults to `client.chain?.id`. Sources are pure reducers (`(tick, prev) → emission`), fully unit-testable without a network. The feed API is `{ watch, unwatch, stop }`, and `createFeedRegistry()` memoizes one feed per `(client, chainId)` so an app never accidentally runs two heartbeats for the same chain. Subscriptions are `useSyncExternalStore`-compatible stores whose snapshot carries the current value, a rolling tick buffer, a delivered-log buffer, staleness, and the last error; the event model is `tick`/`log`/`retraction` (full log)/`gap`/`stale`/`error`, with block-latency reorg self-healing, trailing-window log retraction, and an always-on catch-up window (`maxCatchupBlocks`) that recovers recent logs after a stall or emits a `gap` beyond the bound. v1 ships `pricePathSource` (v2/v3/v4 spot composed via sdk-core `Price`, with failure-isolated leg reads and a `priceFloat`), the launch-lifecycle sources moved in-package (`quickLaunchAssetSource`/`launchAssetSource`/`ccaBidsSource` + `decodeBidSubmitted`, reading descriptors from `@uniswap/liquidity-launcher-sdk`), an on-chain discovery module (`@uniswap/blockfeed-sdk/discovery`: `discoverPricePath`, `isNoPathFound`) that selects by two-way executable quality rather than easily-faked spot or liquidity, and `toCurrency`/`q96ToPrice` helpers. The core references no browser APIs. Supported chains: Ethereum mainnet, Base, and Unichain.
